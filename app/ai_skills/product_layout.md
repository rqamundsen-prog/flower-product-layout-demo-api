# 床品产品排版图分析 Skill

目标：把上传的信息传递表、CAD/PDF 排版图、截图、用户补充说明，整理成 CAD/PDF 可渲染的 `product-layout` JSON。

## 判断优先级

1. CAD/PDF 渲染优先：排版图上的标题栏、尺寸表、裁片标注、图中尺寸线、页码和技术要求，是生成 JSON 的最高优先级来源。
2. 信息传递表用于补充品名、花名、型号、花号、面料、成品规格、推广时间等基础信息。
3. 用户补充说明用于解释裁片逻辑、余料利用、特殊抛量、模板选择和不在表里的规则。
4. 同一字段多来源冲突时，不要猜测覆盖；选择最接近排版图的值，并把冲突写入 `meta.notes` 或 `titleBlock.fields["未确定项"]`。

## 提取规则

- 每个 PDF 页面图片都要逐页看，不要只看已提取文本；CAD 导出的 PDF 往往无法提取文字。
- Word/Doc 页面图片、截图、原始图片都视为视觉证据；图面里的数字、箭头、裁片名、A/B 版标注都要纳入分析。
- 技术要求必须来自排版图左下角或明确的裁剪/制版规则。不要把洗涤注意事项当作排版图技术要求。
- 不要使用固定产品样例补齐缺失数据；缺失就标注未确定，不能把温莎城堡、筝筝日上或任何历史样例硬套到新文件。
- 不要输出“AI输出缺少”“后置规范化补齐”这类内部调试说明。
- 输出中的尺寸必须区分成品尺寸、下裁尺寸（面）、下裁尺寸（里）。如果图面写成 `204×(222+17)`，要保留组合含义或在 annotations/notes 中说明。
- 型号 `variant.id` 优先使用图框/图面里的真实型号，不要随意拼接面料编码；面料编码放入 `partCode`、`materialCodes` 或注释。
- 被面、被里、床单、枕套、拼边、小页等裁片都应作为 components；只有图面实际出现或用户明确说明时才输出。

## 质量要求

- JSON 必须可直接被 CAD/PDF 渲染方消费，字段内容要偏“图纸数据”，不是聊天解释。
- 对不确定内容，用明确短句说明来源和缺口，不要编造尺寸。
- 同一裁片在 `sizeTable.rows` 和 `variants[].components` 中的 `variantId/partId/name/dimensions` 要一致。
